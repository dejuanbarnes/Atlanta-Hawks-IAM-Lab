# Atlanta Hawks IAM Active Directory Lab

## Project Overview

This project is an enterprise-style Identity and Access Management lab built using Windows Server Active Directory Domain Services. The lab models the Atlanta Hawks as a simulated company environment and demonstrates core IAM skills including identity lifecycle management, RBAC, group-based access control, delegated administration, privileged access separation, auditing, and PowerShell automation.

## Technologies Used

- Windows Server
- Active Directory Domain Services
- DNS
- Group Policy Management
- PowerShell
- Windows Client Workstation
- VMware

## Skills Demonstrated

- Active Directory administration
- User provisioning and deprovisioning
- Identity lifecycle management
- RBAC implementation
- AGDLP group strategy
- Security group management
- Shared folder permissions
- Delegated administration
- Privileged account separation
- Group Policy configuration
- Account auditing
- PowerShell automation

## Lab Domain

Domain: `hawks.local`

Domain Controller: `HAWKS-DC01`

Client Workstation: `HAWKS-WS01`

## Business Scenario

The Atlanta Hawks organization is modeled as an enterprise with multiple departments including Players, Coaching Staff, Front Office, Medical Team, Marketing, Finance, and IT. Each department has different access requirements based on job role and business need.

## IAM Scenarios

1. New user onboarding
2. Role change / mover process
3. User offboarding
4. Temporary access request
5. Helpdesk password reset delegation
6. Access review reporting
7. Privileged admin account separation

## Project Outcome

This lab demonstrates practical IAM administration skills by automating user and group management, enforcing least privilege access, validating access control, and documenting identity lifecycle workflows.
